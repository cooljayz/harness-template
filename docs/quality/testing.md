# Testing Strategy

## 피라미드

1. **Unit** — 순수 함수, 타입 변환, 비즈니스 규칙. 빠르고 많이.
2. **Integration** — Repo 어댑터는 실제 DB/외부 서비스(혹은 testcontainer) 대상으로.
3. **Architecture** — `tests/architecture/` 계층 의존성 규칙 검증. 네버 스킵.
4. **E2E** — A 등급 사용자 플로우.
5. **Mutation** — A 등급 핵심 로직. 테스트가 실제로 로직을 잡는지 확인.

## 금지

- Repo 계층을 Service 테스트에서 목킹하는 것: 실제 구현을 in-memory fake으로 대체하라. 목은 구조 변경에 취약하고 에이전트가 잘못된 신호를 받는다.
- 날짜/난수/시스템 시계에 직접 의존: 주입 가능한 clock/random 사용.

## 에이전트를 위한 실패 신호

테스트 실패는 **무엇이** 실패했는지, **어디서** 실패했는지, **왜** 실패했는지를 한 줄 요약으로 포함해야 한다. `assert` 메시지를 구체적으로 작성하라.
